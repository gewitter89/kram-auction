## 2. Tested Locally

*   **Bid Flow:** Валідація кроку ставок, блокування ставок від власного імені, блокування ставок після завершення торгів.
*   **Report Flow:** Створення скарг на лоти користувачами з детальними текстовими коментарями (до 500 символів).
*   **Admin Moderation:** Швидка зміна статусів скарг (`dismissed`, `action_taken`, `reviewed`, `pending`) адміністратором через оновлену адмін-панель.
*   **AuditLog:** Автоматичне ведення системного аудиту дій модерації у базі даних.
*   **Seller Verification:** Можливість ручної верифікації профілю модератором через адмін-кабінет із реактивним оновленням Trust Card на сторінках лотів.
*   **Rate Limit:** Серверне лімітування запитів скарг та ставок з метою захисту від автоматизованого спаму.

---

## 3. Known Limitations

*   **In-Memory Rate Limit:** Локальні ліміти запитів працюють у межах RAM поточного інстансу.
*   **In-Memory SSE / EventBus:** Сповіщення реального часу про зміну цін працюють у пам'яті сервера без підтримки горизонтального масштабування.
*   **No Production Redis:** Відсутній єдиний розподілений кеш стану лімітів та черг подій.
*   **No Production-Grade Realtime Provider:** Не підключено комерційні сервіси реального часу (на кшталт Ably чи Pusher).
*   **No LiqPay / TurboSMS:** Всі платежі та зв'язок на даному етапі є прямими домовленостями сторін без залучення сторонніх шлюзів у релізі.
*   **Controlled Beta Only:** Проект позиціонується як закритий бета-тест без автоматизованого ескроу-сервісу.
